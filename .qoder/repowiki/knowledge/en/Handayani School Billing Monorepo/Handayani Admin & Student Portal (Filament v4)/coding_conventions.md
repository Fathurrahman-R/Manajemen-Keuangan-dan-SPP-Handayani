- Every Livewire table component fetches paginated records by calling `ApiService::client()->get(...)` and wraps the response in a `LengthAwarePaginator`, never using Eloquent queries directly.
- API calls in Livewire components are wrapped in try/catch blocks that delegate failures to `HandlesApiErrors::handleApiError()`, `notifyConnectionError()`, or `notifyUnexpectedError()` rather than surfacing raw exceptions.
- Sidebar navigation items are constructed with `NavigationItem::make()` and gated by `PermissionHelper::has('...')` / `PermissionHelper::hasAnyInGroup('...')` checks, with `isActiveWhen` comparing against `original_request()->routeIs('filament..pages.<route>')`.
- Jenjang-based sub-navigation (KB/TK/MI) is generated by iterating `NavigationConfig::JENJANG_OPTIONS` and appending `?jenjang=<code>` query parameters to page URLs.
- Panel branding (brandName, brandLogo, favicon, colors) is resolved through dedicated `resolveBrand*` methods that read `BrandingService::get()` and fall back to `env('APP_NAME', 'Handayani')` when no custom branding is configured.
- Logout actions in both panels follow the same pattern: attempt `ApiService::client()->delete('/logout')`, then `Filament::auth()->logout()` plus `session()->flush()/invalidate()/regenerateToken()` before redirecting to the login URL.
