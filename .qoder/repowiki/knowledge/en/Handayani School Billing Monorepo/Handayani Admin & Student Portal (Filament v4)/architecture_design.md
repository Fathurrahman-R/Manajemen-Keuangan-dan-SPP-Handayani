Two independent Filament panels are registered via `app/Providers/Filament/AdminPanelProvider.php` (default panel at root path) and `PortalPanelProvider.php` (under `/portal`). Each provider configures its own login, middleware stack, branding (logo/favicon/colors resolved through `BrandingService`), SPA mode, breadcrumbs, and navigation. The Admin panel uses a permission-driven sidebar built in `AdminPanelProvider::buildNavigation()` that groups items into Akademik / Keuangan / Laporan / Pengaturan according to `App\Config\NavigationConfig`, with per-jenjang (KB/TK/MI) sub-items for Siswa/Kelas/Tagihan/KenaikanKelas. Pages live under `Filament/Pages/*` (admin) and `Filament/Portal/Pages/*` (student/parent); views are Blade templates under `resources/views/filament/pages`. Interactive data screens are implemented as Livewire components (`app/Livewire/*.php`) implementing `HasTable`/`HasActions`/`HasSchemas`, which call the remote backend exclusively through `App\Services\ApiService::client()` — no Eloquent models are used inside this module. Shared concerns (API error handling, import/export, period filter) are extracted into `Livewire/Concerns/*`. Widgets (`Filament/Widgets/*`) render dashboard charts via Chart.js. Authentication is delegated to a custom `Http/Middleware/CustomAuthentication` that reads session data from the API; logout actions also invoke `ApiService::client()->delete('/logout')` before clearing the local session. Feature toggles (`handayani.features.*`) in `config/handayani.php` gate Midtrans, SPA loading, custom navigation, and portal visibility.
