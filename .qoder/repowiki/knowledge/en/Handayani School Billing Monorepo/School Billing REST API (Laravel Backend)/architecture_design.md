Standard Laravel 12 layout under `backend/` with PSR-4 autoloading (`App\`, `Database\Factories\`, `Database\Seeders\`). The HTTP layer is thin: controllers in `app/Http/Controllers/*` accept typed Request classes from `app/Http/Requests/*`, enforce per-endpoint authorization via `permission:*` middleware on `routes/api.php`, and delegate business logic to single-responsibility services in `app/Services/*`; responses are shaped through Eloquent Resources in `app/Http/Resources/*`. Domain models live in `app/Models/` backed by migrations/seeders/factories under `database/`. Cross-cutting concerns follow Laravel conventions: Events (`PembayaranRecorded`, `TagihanCreated`) → Listeners (`SendKwitansiNotification`, `SendTagihanBaruNotification`) + Notifications for async workflows; queued Jobs (`ProcessExportJob`, `ProcessImportJob`) for import/export; Console Commands for scheduled tasks; Observers (`SiswaObserver`, `DashboardCacheObserver`) for model hooks; feature-scoped Exceptions grouped under `app/Exceptions/<Feature>/` (notably `Midtrans/*`).

The Midtrans payment subsystem is isolated behind the `Services/Midtrans/MidtransClient` interface plus dedicated services (`MidtransInitiationService`, `MidtransStatusSyncService`, `SignatureVerifier`, `StatusMapper`, `OrderIdGenerator`, `MidtransFeeService`, `MidtransLogService`) so the gateway can be toggled off via `config('midtrans.enabled')` without redeploy. Import/export uses `maatwebsite/excel` with a two-phase upload→preview→confirm flow orchestrated by `Services/ImportExport/*` and persisted through `ImportBatch`/`ExportJob` models. Authentication uses Sanctum tokens with abilities derived from `spatie/laravel-permission` v7; route-level authorization is declared at the routing layer. API docs are auto-generated at runtime by `dedoc/scramble` and static OpenAPI specs are committed under `docs/*.json`.
