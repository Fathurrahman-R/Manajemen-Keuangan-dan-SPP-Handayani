- Entry points: `src/start.ts` bootstraps the TanStack Start server via `createStart` with an error-handling request middleware; `src/server.ts` is the Nitro/Cloudflare Worker entry that wraps the generated server-entry to normalize h3-swallowed SSR errors into HTML pages.
- Routing: file-based routes under `src/routes/` (`__root.tsx`, `index.tsx`) using `createRootRouteWithContext` and `createFileRoute`; route tree is code-generated at `src/routeTree.gen.ts` by the TanStack Router plugin.
- Data layer: a single `QueryClient` instance created in `src/router.tsx` and injected as router context, then provided via `<QueryClientProvider>` in the root component — no per-route client.
- Component layers: `src/components/ui/` holds shadcn/ui primitives (Radix + Tailwind), `src/components/sections/` composes page sections (Nav, Hero, About, Jenjang, SppCta, Kontak, Footer) consumed by the index route, plus shared `GeometricPattern` and `Reveal` animations.
- Styling: Tailwind v4 via `@tailwindcss/vite` with CSS variables (base color `slate`); global styles live in `src/styles.css` and are imported through the root route's `head.links`.
- Build/dev: Vite config delegates to `@lovable.dev/vite-tanstack-config` which auto-wires tanstackStart, nitro (cloudflare target), tsConfigPaths, and the `@/*` alias; `bunfig.toml` pins Bun as the runtime.
- Error reporting: `src/lib/error-capture.ts` captures unhandled exceptions, `src/lib/lovable-error-reporting.ts` forwards them to Lovable, and `src/lib/error-page.ts` renders a fallback HTML page used by both server and start middleware.
