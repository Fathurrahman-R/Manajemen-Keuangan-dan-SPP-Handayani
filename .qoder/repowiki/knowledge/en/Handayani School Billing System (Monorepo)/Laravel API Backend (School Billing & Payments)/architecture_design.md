Standard Laravel 12 project layout under `backend/` with PSR-4 autoloading (`App\`, `Database\Factories\`, `Database\Seeders\`). HTTP layer is split into thin controllers in `app/Http/Controllers/*` that delegate business logic to domain services in `app/Services/*`; request validation lives in `app/Http/Requests/*` and response shaping in `app/Http/Resources/*`. Domain models sit in `app/Models/` backed by migrations/seeders/factories under `database/`. Cross-cutting concerns are organized as: Events/Listeners/Notifications for async workflows (e.g. `PembayaranRecorded` → `SendKwitansiNotification`), Jobs (`ProcessExportJob`, `ProcessImportJob`) queued via Laravel queue, Console Commands for scheduled tasks, Observers for model hooks, and custom Exceptions grouped by feature (notably `Exceptions/Midtrans/*`). The Midtrans payment subsystem is isolated behind the `Services/Midtrans/MidtransClient` interface plus dedicated services (`MidtransInitiationService`, `MidtransStatusSyncService`, `SignatureVerifier`, `StatusMapper`, `OrderIdGenerator`, etc.) so the gateway can be toggled off via `config('midtrans.enabled')` without redeploy. Authentication uses Sanctum tokens with abilities derived from `spatie/laravel-permission` roles; route-level authorization is enforced per-endpoint via `permission:*` middleware on `routes/api.php`. Import/export is built on `maatwebsite/excel` with a two-phase upload→preview→confirm flow orchestrated by `Services/ImportExport/*` and persisted through `ImportBatch`/`ExportJob` models. API docs are generated at runtime by `dedoc/scramble` and static OpenAPI specs live in `docs/*.json`.
