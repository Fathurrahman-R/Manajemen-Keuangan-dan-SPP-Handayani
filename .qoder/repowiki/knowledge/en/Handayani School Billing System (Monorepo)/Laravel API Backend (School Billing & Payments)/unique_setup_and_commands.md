Composer scripts provide one-shot bootstrap: `composer run setup` installs deps, copies `.env.example` to `.env`, generates app key, runs migrations, then builds assets; development mode runs three concurrent processes via `composer run dev` (server, queue worker listening with `--tries=1`, Vite dev server). Tests are invoked through `composer run test` which clears config first. A helper script `reset_test_db.php` exists for wiping the test database.
